Perancangan Sistem Informasi Pengarsipan Akta Nikah

KUA Kemantren Tegalrejo - Laravel 12

Ringkasan Proyek

Sistem web untuk mengelola arsip akta nikah dengan tiga peran pengguna: Admin, Administrator Data, dan Kepala KUA. Sistem sudah memiliki fondasi dari proyek existing, termasuk autentikasi dan modul arsip dasar.

Rancangan Database (ERD)

USERS
id - bigint PK
name - string
email - string UK
password - string
role - enum
is_active - boolean
created_at - timestamp
updated_at - timestamp
ARSIP
id - bigint PK
nomor_akta - string UK
tanggal_akta - date
nama_suami - string
nama_istri - string
lokasi_kua - string
keterangan - text
file_path - string
user_id - bigint FK
created_at, updated_at
LOG_PENCARIAN
id - bigint PK
user_id - bigint FK
kata_kunci - string
waktu - timestamp

Relasi: USERS (1) ─── (∞) ARSIP | USERS (1) ─── (∞) LOG_PENCARIAN

Perubahan Tabel

  • users: Ubah enum role menjadi admin, admin_data, kepala_kua. Tambah kolom is_active.
  • arsip: Rename tanggal_pernikahan → tanggal_akta, lokasi_fisik → lokasi_kua. Tambah keterangan.
  • log_pencarian: Tabel baru untuk mencatat histori pencarian.

Catatan Penting

⚠️ Perubahan Schema Database
Migration akan mengubah struktur tabel arsip (rename kolom, hapus kolom redundan). Pastikan backup data sebelum menjalankan migration.
🔴 Existing Data
Jika ada data arsip existing, kolom tahun_pernikahan dan kategori akan dihapus. Pastikan data sudah di-backup.
